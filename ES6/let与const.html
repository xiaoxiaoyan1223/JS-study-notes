var let 声明的变量可以进行重新赋值 const声明的变量相当于常量 不可以进行重新赋值

let与const的相同点 1、let和const的相同点： ①只在声明所在的块级作用域内有效。 ②不提升，同时存在暂时性死区，只能在声明的位置后面使用。 ③不可重复声明。

let与const的不同点 ①let声明的变量可以改变，值和类型都可以改变；const声明的常量不可以改变，这意味着，const一旦声明，就必须立即初始化，不能以后再赋值。 ②数组和对象等复合类型的变量，变量名不指向数据，而是指向数据所在的地址。const只保证变量名指向的地址不变，并不保证该地址的数据不变，所以将一个复合类型的变量声明为常量必须非常小心。 const arr = []; // 报错，[1,2,3]与[]不是同一个地址 arr = [1,2,3]; const arr = []; // 不报错，变量名arr指向的地址不变，只是数据改变 arr[0] = 1; arr[1] = 2; arr[2] = 3;
